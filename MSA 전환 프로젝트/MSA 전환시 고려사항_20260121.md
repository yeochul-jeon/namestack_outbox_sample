## 2. 아키텍처 및 기술 관점

## 2.1. 서비스 분할 전략

- **고려사항**
    - 서비스 경계(도메인) 설정의 어려움
    - 과도한 세분화로 인한 복잡성 증가
        
- **해결 방법**
    - 비즈니스 로직 중심의 서비스 경계 설계
    - Event Storming 등 도메인 분석 기법 활용
    - 우선적으로 큰 단위로 분리 후 점진적으로 세분화(Go Macro, then Micro)[4](https://velog.io/@sorzzzzy/MSA-MSA-%EC%84%9C%EB%B9%84%EC%8A%A4-%EB%B6%84%EB%A6%AC-%EC%9B%90%EC%B9%99-%EB%B0%8F-%EA%B3%A0%EB%A0%A4%EC%82%AC%ED%95%AD)[1](https://www.msap.ai/docs/msa-expert-from-concepts-to-practice/part-1-msa-fundamentals/chapter-1-introduction-to-msa/section-1-3-considerations-for-adopting-msa/)[2](https://www.msap.ai/docs/msa-expert-from-concepts-to-practice/part-1-msa-fundamentals/chapter-1-introduction-to-msa/section-1-3-considerations-for-adopting-msa/subsection-1-3-2-transition-strategies-and-step-by-step-approach/)
        
## 2.2. 데이터베이스 및 데이터 관리

- **고려사항**
    - 데이터베이스 공유로 인한 결합도 증가
    - 데이터 일관성 및 분산 트랜잭션 문제
- **해결 방법**
    - 서비스별 독립 데이터베이스 원칙 적용
    - 데이터 마이그레이션 및 동기화 전략 수립
    - eventual consistency, SAGA 패턴 등 분산 트랜잭션 처리 방안 도입[4](https://velog.io/@sorzzzzy/MSA-MSA-%EC%84%9C%EB%B9%84%EC%8A%A4-%EB%B6%84%EB%A6%AC-%EC%9B%90%EC%B9%99-%EB%B0%8F-%EA%B3%A0%EB%A0%A4%EC%82%AC%ED%95%AD)[5](https://yummy0102.tistory.com/367)[6](https://danmilife.tistory.com/38)
        

## 2.3. 서비스 간 통신 및 API 설계
- **고려사항**
    - 서비스 간 통신 방식 및 API 표준화 필요
    - 네트워크 장애, 지연 등 분산 환경의 복잡성
- **해결 방법**
    - RESTful API, gRPC 등 적합한 통신 방식 선택
    - API Gateway 및 서비스 디스커버리 도입
    - Circuit Breaker, Retry 등 장애 대응 패턴 적용[1](https://www.msap.ai/docs/msa-expert-from-concepts-to-practice/part-1-msa-fundamentals/chapter-1-introduction-to-msa/section-1-3-considerations-for-adopting-msa/)[6](https://danmilife.tistory.com/38)
        
## 3. 인프라 및 운영 관점

## 3.1. 배포 및 운영 자동화

- **고려사항**
    - 서비스별 독립적 배포 체계 필요
    - 운영 및 장애 대응 복잡성 증가
        
- **해결 방법**
    - CI/CD 파이프라인 구축
    - 컨테이너 오케스트레이션(Kubernetes 등) 도입
    - 자동화된 모니터링, 로깅, 알람 시스템 구축[1](https://www.msap.ai/docs/msa-expert-from-concepts-to-practice/part-1-msa-fundamentals/chapter-1-introduction-to-msa/section-1-3-considerations-for-adopting-msa/)[6](https://danmilife.tistory.com/38)
        
## 3.2. 보안 및 인증
- **고려사항**
    - 서비스별 개별 보안 정책 필요
    - 인증/인가, 데이터 암호화 등 보안 강화 요구
- **해결 방법**
    - 중앙 집중형 인증(SSO, OAuth 등) 및 API 보안 적용
    - 서비스별 취약점 점검 및 보안 정책 수립
