# 작업할 사항
* bff 리팩토링
  * GlobalExceptionHandler 리팩토링 - 진행중
    * 일관성 있는 로그 포맷 적용 
  * virtual thread 설정 관리 방안 고민 - 종료
    * 별도의 bean 설정을 통해 관리 되기 때문에 다른 모듈에 영향은 없을 것으로 보임
    * 다만 사용하는 측에서 @Qualifier를 통해 구분하여 사용해야 함 - 점검 결과 이상없을 것 같음
  * VirtualThreadConfig 네이밍 변경 - 완료
    * VirtualThreadConfig -> BffVirtualThreadConfig
      * BFF 모듈에서만 사용하기 때문에 중복을 피하기 위해 변경
  * boundedVirtualExecutor 빈 이름 변경 - 진행중
    * bffBoundedVirtualExecutor -> bffBoundedVirtualExecutor
      * 빈 이름 중복 방지
    * 사용법 문의 및 개선 방안 검토
