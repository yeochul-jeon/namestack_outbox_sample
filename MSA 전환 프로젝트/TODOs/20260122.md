# Git 브랜치 기본 이름 변경 (main → master) 작업절차 정리

## 변경 방법

**로컬 브랜치 이름 변경:**
```bash
git branch -m main master
```

**원격 저장소에 변경사항 반영:**
```bash
git push -u origin master
```

**원격 저장소에서 기존 main 브랜치 삭제:**
```bash
git push origin --delete main
```

## 주의사항

1. **팀 협업 환경**: 다른 팀원들도 각자 로컬에서 같은 작업을 수행해야 합니다.
   ```bash
   git fetch origin
   git checkout master
   git branch -d main
   ```

2. **기본 브랜치 설정**: GitLab 웹 인터페이스에서 저장소 설정(Settings > Repository)으로 이동하여 기본 브랜치를 `master`로 변경해야 합니다.

3. **CI/CD 파이프라인**: `.gitlab-ci.yml` 등에서 `main` 브랜치를 참조하는 부분이 있으면 `master`로 수정하세요.

4. **Pull Request/Merge Request**: 진행 중인 MR이 있다면 마지막에 변경하거나 재생성해야 합니다.

5. **보호된 브랜치**: GitLab에서 `main` 브랜치가 보호되어 있다면 먼저 보호를 해제한 후 변경해야 합니다.
