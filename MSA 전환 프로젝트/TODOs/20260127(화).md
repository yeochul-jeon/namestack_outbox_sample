## Git에서 remote의 파일 구조 변경사항을 로컬에 완전히 반영하려면 다음 명령어를 사용

~~~bash
git fetch origin develop
git reset --hard origin/develop
git clean -fd
~~~

각 명령어의 역할:
* git fetch origin develop: 원격 develop 브랜치의 최신 상태를 가져옵니다
* git reset --hard origin/develop: 로컬을 원격 develop과 완전히 동일하게 맞춥니다 (로컬 변경사항 삭제)
* git clean -fd: Git이 추적하지 않는 파일과 디렉토리를 삭제합니다 (삭제된 패키지의 잔여 파일들 제거)

---

>현재 develop pull 을 수행했습니다. 그런데 remote devleop 에서 패키지 구조를 크게 변경하여 local 에 기존 패키지 들이 남아있는데 이를 어떻게 정리하고, 
>develop 기준으로 맞출수 있나요?

## Git에서 원격 브랜치의 패키지 구조 변경을 로컬에 반영하려면 다음 방법들을 사용할 수 있습니다:

1. 가장 안전한 방법 (추천)
현재 작업 중인 변경사항이 있다면 먼저 백업하고:
~~~bash
git clean -fd
~~~
이 명령은 추적되지 않는 파일과 디렉토리를 삭제합니다.


2. 더 철저한 정리가 필요한 경우
~~~bash
git clean -fdx
~~~
-x 옵션은 .gitignore에 명시된 파일도 함께 삭제합니다 (빌드 산출물, IDE 설정 등).


3. 실행 전 미리 확인하기
~~~bash
git clean -fdn
~~~
-n 옵션으로 실제로 삭제되지 않고 어떤 파일들이 삭제될지 미리 볼 수 있습니다.

주의사항
* 이미 develop pull을 수행했다면, Git이 추적하는 파일들은 자동으로 삭제되었을 것입니다
* 남아있는 것은 Git이 추적하지 않는(untracked) 파일들입니다
* 중요한 로컬 작업이 있다면 먼저 백업하세요
